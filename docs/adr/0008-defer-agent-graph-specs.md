# Defer Agent Graph Specs in favor of TypeScript LangGraph factories

Do not introduce a language-neutral Agent Graph Spec package yet. Agent graph behavior will be defined by TypeScript LangGraph factories, because using LangGraph directly preserves the full runtime API while the project is still learning which graph variations matter.

Python can still manage trials by selecting graph factories, generating parameters, invoking the TypeScript runtime, and collecting results. It should not require a separate graph definition language while graph code, prompt files, tool code, and runtime code all live in Git.

Reproducibility should use Git as a source checkout mechanism for now, not as the complete identity of agent behavior. Each published graph version must bind to a clean Source Revision so engineers can recover the graph factory, prompt files, tool code, and runtime code, but the behavior identity also needs to include selected graph, state, action, prompt, tool, model, and parameter versions. These dimensions should be recorded as an Agent Behavior Version tuple in Diagnostic Telemetry rather than collapsed into a single hash, so trials can be queried by constraints such as one state version across multiple action versions. Strict published or production enforcement should be selected by a validated Runtime Profile rather than inline environment-variable policy: every tuple dimension must resolve before runtime, and missing, `none`, or `unknown` values are invalid version states that should be rejected before execution and surfaced as telemetry errors. Runtime Profile schemas should live in `packages/shared`, and default profile documents should live under a top-level `profiles/` directory. Development Runtime Profiles may allow incomplete local ad hoc Agent Runs, but those runs are not comparable or promotable. The no-dirty-worktree rule belongs at the trial publishing or graph-version registration boundary. A separate `packages/graph-spec` package remains available later if repeated graph variants reveal a stable implementation-neutral shape worth standardizing.
